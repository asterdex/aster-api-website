## **2026-04-28**

**合约 V3 及测试网 — 新增子账户接口**

- 在合约 V3 及合约测试网中（中英文）新增 4 个子账户接口：
  - `POST /fapi/v3/createSubAccount` — 创建子账户（TRADE），需要两次独立签名（子账户 `childSignature` + 母账户 `signature`）
  - `GET /fapi/v3/getSubAccountList` — 查询子账户列表（USER_DATA），无论母账户为 EVM 还是 Solana 地址，始终使用 signer 私钥（EIP-712）签名
  - `POST /fapi/v3/updateSubAccount` — 更新子账户（TRADE），支持修改子账户名称及 `NORMAL` / `FROZEN` 状态切换
  - `POST /fapi/v3/subAccountTransfer` — 子账户划转（TRADE），支持母账户与子账户之间的 `FUTURE_FUTURE`、`FUTURE_SPOT`、`SPOT_FUTURE`、`SPOT_SPOT` 划转

**合约 V3、合约测试网、现货 V3 及现货测试网 — 鉴权章节更新**

- 鉴权说明拆分为两种独立场景（中英文）：
  - **API_WALLET 鉴权** — 在请求体中添加 `signer`
  - **主地址鉴权** — 在请求体中添加 `user`
- 从 `send_by_url` 和 `send_by_body` Python 签名示例中移除 `my_dict['user'] = user`（`user` 字段不再是签名必需字段）

**链上 RPC — 现货接口新增及当前挂单更新**

- 新增 2 个 RPC 接口（中英文）：
  - `aster_spotOpenOrders` — 查询用户现货当前挂单，仅返回不早于 block 1 创世时间的挂单，最多 1000 条
  - `aster_spotUserFills` — 查询用户现货成交记录，可按时间范围查询（最多 7 天，最多 1000 条）
- 更新 `aster_openOrders`（查询当前挂单）的返回结构（中英文）：
  - 在示例和字段表中新增 `openOrders[].price`（委托价格）字段
  - 在示例和字段表中移除 `openOrders[].status`（如 `NEW`）字段
