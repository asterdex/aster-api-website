## **2026-05-20**

**Futures V3 与测试网 — 追单接口与 BBO peg 订单支持**

- 在 Futures V3 与 Futures Testnet 的 `POST /fapi/v3/order` —— **下单接口** 中新增 3 个可选参数（EN & ZH）：
  - `pegPriceType` (ENUM) —— BBO peg 模式：`COUNTERPARTY_1` 或 `QUEUE_1`。在 `LIMIT` 订单上设置此参数后，撮合引擎在触发时基于订单簿的 BBO 加 `pegOffset` 解析实际价格。默认不使用 peg。
  - `pegOffset` (DECIMAL) —— 当 `pegPriceType` 已设置时，相对 BBO 的有符号偏移量。买单：非正值（如 `-0.5`）；卖单：非负值。必须为 `tickSize` 的倍数。
  - `priceLimit` (DECIMAL) —— BBO peg 订单的绝对价格上下限。买单：上限；卖单：下限。必须 > 0 且为 `tickSize` 的倍数。默认无限制。

- 在 Futures V3 与 Futures Testnet 新增接口 `POST /fapi/v3/chase` —— **追单 (TRADE)**（EN & ZH），位于 `修改订单` 与 `批量下单` 之间：
  - 下一笔 **BBO peg GTX 限价单，自动跟随价格** —— 策略服务每个 tick 轮询订单簿，实时修改委托价为 `bid1 − chaseOffset`（买单）或 `ask1 + chaseOffset`（卖单）。
  - 支持 `quantityUnit`（`BASE` / `QUOTE`）、`chaseOffset` / `chaseOffsetType`、`maxChaseOffset` / `maxChaseOffsetType`（`ABSOLUTE` / `PERCENTAGE`）以及 `priceLimit`。
  - 当市场偏移超过 `maxChaseOffset` 时**自动撤销**，原因为 `OFFSET_CANCELLED`；当新 peg 价格突破 `priceLimit` 时 clamp 在 `priceLimit`。
  - `timeInForce` 默认 `GTX`（post-only）；不允许 `NO_FILL`。
  - `clientStrategyId` 长度 ≤ 28 字符，须满足 `^[\.A-Z\:/a-z0-9_-]{1,28}$`。

- 更新 Futures V3 与 Futures Testnet **修改订单** 说明（EN & ZH）：
  - 增加注意事项：使用 `pegPriceType = COUNTERPARTY_1 / QUEUE_1` 的 BBO peg 订单无法通过普通 modify 改变 peg 解析后的价格；若需要持续追踪 BBO，请使用追单接口 `POST /fapi/v3/chase`。
